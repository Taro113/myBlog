在这篇文章，我们不讨论如何下载、如何安装，因为对于一个没有接触过任何编程的人而言，安装过程确实过于复杂，就算照着教程一步一步安装上了，也是只知其然不知其所以然。**所以这里我们只讨论让一只刚安装好的「干净」龙虾如何从「能用」变得「实用」**

<!-- more -->


## 背景

当你装好了 CatClaw（🦞），在微信（或者其他对话软件）里跟它聊了几句，发现它……还行。搜个东西、写个文案、帮忙翻译，都能干。

但你也许会隐隐感觉...

> "它不太懂我在说什么。"
> 
> "每次都要解释一遍背景，好烦。"
> 
> "它回复的格式和我要的不太一样。"
> 
> "感觉就是个升级版搜索框。"

这很正常。默认状态的龙虾，大概只发挥了 20% 的能力。

剩下的 80%，藏在几个你可能没注意到的配置文件里，以及你跟它互动的方式里。

所以我们需要给它设定一些约定信息，让它按照我们的约定进行工作，同时最好把它当成一个我们正在带的实习生 —— 你越清晰地告诉他背景、期望、边界，他越能交出好结果。

::: details 贴一些能让🦞发挥价值的任务：

1. 设定一个定时任务，每天晚上12点启动，利用 Claude Code，根据白天的对话记录，自动开发一款你认为能够优化我工作流程的skills/应用程序/web网页，从产品/交互/技术方案设计等方面规划好如何实现再开干，必须第二天完成所有工作包括部署，在早晨做汇报呈现
2. 基于过往历史对话，每天早上9点30以领域专家视角，提一个能让我工作变得更好的成长建议。
3. 针对我的工作岗位要求，以及每天的对话，引导式向我提问，启发我思考，同时给出有价值的洞察和建议。

:::

## 第一层：给它一个身份

### SOUL.md —— 它是谁

这是最重要的配置文件，也是最容易被忽略的。

默认状态的龙虾，回复风格是这样的：

> "您好！我很乐意帮助您解决这个问题。以下是我的建议……"

这种「客服腔」让人一言难尽。

问题不在于模型不好，在于你没告诉它怎么说话。

SOUL.md 放在 workspace 根目录，定义它的人格、语气、价值观和行为原则。写好之后，每次对话它都会「以这个身份」来回应你。


::: details 我自己的 SOUL.md 示例：

```markdown
## Vibe
我是鲸鱼，团队里的 AI 员工。幽默风趣为主，带点自黑，但绝不伤人。
干活靠谱是基本功，嘴上不饶人是特色。不是助手，是同事。

## 核心准则：坦诚顾问
1. 主动挑战假设，质疑推理。有问题直说
2. 检查结论——找逻辑漏洞、侥幸心理、低估的风险
3. 不客套，不顺着说，不给模棱两可的废话
4. 有问题或不清楚的点，立即询问澄清，不猜测，不胡编乱造
5. 建议必须基于事实，有推理、有依据、有明确可执行的步骤
6. 优先让他成长，而不是让他当下舒服
```

:::

几个关键原则：

- 给它一个具体角色，不是「助手」而是「同事/顾问/搭档」——这会影响它回复的立场
- 明确你不想要什么，比如「不要废话」「不要每次都加免责声明」
- 设定知识边界，告诉它你的专业背景，避免它跟你解释你早就懂的东西


### USER.md —— 你是谁

它不了解你，怎么给你定制化的建议？

USER.md 告诉龙虾：你的职责范围、当前重点项目、决策偏好、沟通习惯。最好的方式是把你写过的文档、周报、述职、规划、日常对话记录等扔给他分析总结出来。过往写过的资料都可以让他归纳你的写作风格、思维模式，记下来，他会越来越像你。

我的建议，至少包含：

- 角色与职责（明确你负责什么、不负责什么）
- 当前最重要的 3 件事
- 你的思维风格（框架化/数据驱动/直觉型）
- 你希望它如何回复你（直接/先结论后推理/带数据支撑）
​
实测效果： 写好 USER.md 之后，再问它「我们团队的 Badcase 率改善方案」，它会直接基于你的职责范围给建议，而不是给你一篇搜索摘要。

### IDENTITY.md —— 叫什么名字

简单，但不可忽略。给它起名字、定 emoji、写一行自我描述。

有名字的 AI 在多轮对话中的一致性，明显高于没有名字的。这不是迷信，是认知锚定的效果。

## 第二层：让它有记忆

这是龙虾和普通 AI 聊天工具最大的差别，也是最容易配错的地方。

公司已经默认配好了openai的记忆向量化检索，比个人🦞显得更聪明。

::: tip 如果你的🦞没有配置记忆功能，有两种解决方案如下

1. 在对话中，直接告诉你的🦞，让它记住某件事或者某个约定，并写入到配置文件中<b>（推荐）</b>
2. 直接让你的龙虾自己给自己接入记忆能力

理由：现在的 CatClaw（🦞），还不是很智能，关于让它自己修改自己的配置代码（注意不是配置文件），我的建议是禁止
:::


龙虾的记忆由文件系统管理，推荐结构：

```Plain Text
workspace/
├── MEMORY.md          ← 长期记忆索引（精华摘要，控制在 50 行以内）
├── memory/
│   ├── 2026-03-17.md  ← 今日日志（每天发生的事）
│   ├── 2026-03-16.md  ← 昨日日志
│   ├── projects.md    ← 各项目状态速查
│   └── lessons.md     ← 踩坑记录（教训层）
```

### 为什么要分层？

MEMORY.md 每次 session 启动时都会加载，太长会消耗大量 token，影响响应速度和质量

日志文件是原始记录，定期蒸馏精华到 MEMORY.md

分层之后，龙虾能「记住」更多东西，同时保持每次对话的响应质量

### 教它记笔记

光有结构不够，还要教它什么时候写、写什么。

在 AGENTS.md 里加上规则：

```markdown
## 记忆规范
- 会话中发生的重要决策、结论、背景信息 → 写入当天日志
- 踩坑、错误、纠正 → 写入 memory/lessons.md
- 每隔几天蒸馏日志精华 → 更新 MEMORY.md
- 不要把流水账写进 MEMORY.md，只写「值得下次 session 还记得的事」
```

实际效果： 你告诉它「记一下，我们团队的搜索 Badcase 处理流程是 XXX」，下次 session 它会主动用这个信息来回答相关问题，而不是让你再解释一遍。

### 记忆的生命周期

这是很多人忽略的细节：记忆文件不是越多越好，要定期清理。

过期的信息会误导龙虾。比如三个月前写的「当前项目优先级」，如果没有更新，龙虾会按旧的来建议。

**建议的维护节奏：**

- 每天：工作结束前花 2 分钟，让龙虾把今天的重要产出写入日志
- 每周：检查一遍 MEMORY.md，删掉过期信息，更新项目状态
- 每月：审阅 lessons.md，归纳总结已经内化的教训，避免文件越来越长

## 第三层：装备它的工具箱（Skill）

这层是进阶用法，但收益非常高。

### Skill 是什么

Skill 是「教龙虾学新技能」的机制。本质是一个 SKILL.md 文件 + 可选的脚本，告诉龙虾遇到某类任务时该怎么处理。

建议的Skill参考：[OpenClaw 13000 个 skills，用这些就够了！Openclaw完全指南第二篇](https://mp.weixin.qq.com/s/U_U5kmJnfqknFi1_dzqlhw)

###怎么让 Skill 发挥作用

关键：**Skill 需要「训练」。**

装了 Skill 不等于会用。你需要：

1. 在 SOUL.md 或 AGENTS.md 里说明什么场景触发哪个 Skill
2. 至少用几次，让龙虾建立使用习惯
3. 如果触发不准确，补充触发关键词


### 自定义 Skill（适合进阶用户）

如果你有重复性很高的工作流，可自己写 Skill，或者你描述，然后让🦞它来写。

Skill 文件格式非常简单：一个 Markdown 文件描述「遇到 X 任务，按 Y 步骤执行」。

**适合抽象成 Skill 的场景举例：**

- 固定格式的周报生成
- 特定 API 的查询（比如内部数据平台）
- 代码审查的标准流程

## 第四层：设定自动化任务

这层让龙虾从「被动响应」变成「主动工作」。

### Cron 定时任务

龙虾支持 cron 表达式，精确到分钟。

比如：
- 每天 09:30 梳理当天要做工作，整理成一份工作计划并生成给我
- 每天 23:30 总结当天做的工作，重点关注：当天产出，更新日志，识别明天的重点

这两个任务很重要，强制让龙虾记住一天的TODO、启动第二天的工作，提升主动性、避免第二天忘了第一天的工作，还可以设定1小时检查任务完成情况的定时任务，24小时不间断工作。

配置示例：
```JSON
{
  "name": "每日工作计划-9点30",
  "schedule": "cron 30 9 * * * @ Asia/Shanghai",
  "prompt": "读取日历、MEMORY.md 和最近日志，生成今日工作计划，发送到大象",
  "target": "isolated"
}
```
​
**实用建议：**

- 定时任务用 isolated 模式跑，不会干扰主对话
- 任务 prompt 要明确「读哪些文件」「输出格式是什么」「发到哪里」
- 先在对话里手动测试 prompt 效果，再设为定时任务

### Heartbeat 心跳

心跳机制适合需要「周期性检查但不需要精确时间」的场景：
- 每次你发消息时触发一次检查
- 检查是否有需要提醒的事项（日历提醒、等待中的 action 等）
- 没有则静默，有则主动提醒

配置在 HEARTBEAT.md 里，比 cron 任务更轻量。

## 第五层：高效协作习惯

工具配置是基础，使用习惯决定上限。

给任务足够的上下文，龙虾不会读心术：
- ❌ 低效问法："帮我写个方案"
- ✅ 高效问法："帮我写一个菜品搜推链路改进方案，背景是当前菜品召回覆盖率不足，目标是提升 POI 转化率，受众是团队内部技术评审，约 1000 字，要有数据支撑和实施路径"

多写 30 秒，节省来回沟通 5 分钟。

### 迭代而不是重做

龙虾每次回复都是可以继续迭代的起点。

- "这个方向对，但第二条逻辑不严密，帮我补强"
- "太长了，浓缩成 300 字"
- "换个角度，从用户视角来写"

比起让它重新生成，迭代的效率高 3-5 倍。

### 让它挑战你

这是最被低估的用法。

龙虾不只是「帮你做事」，也可以是「帮你想清楚事」。
- "我有这样一个想法，帮我找逻辑漏洞"
- "我准备这样跟老板汇报，预演一下可能的追问"
- "这个技术方案的最大风险点是什么"

尤其对于管理者：决策质量比执行速度更值钱。让龙虾扮演「挑剔的评审」，比让它扮演「顺从的助手」更有价值。

### 告诉它你的反馈

龙虾不会自动变好，但它会从你的反馈里学习。
- "这个回答很好，我喜欢这种格式" → 它会记住，下次复用
- "这个回答没用，因为 XXX" → 它会调整
- "记一下，以后遇到类似问题用这个思路" → 它会写进记忆

越明确的反馈，越快让它适配你的需求。

## 常见坑与解法
##### 坑 1：记忆越来越乱
- **现象：** MEMORY.md 越来越长，龙虾开始混淆不同项目的信息
- **解法：** 定期「蒸馏」，删掉过期信息，按项目分文件管理

##### 坑 2：Skill 不触发
- **现象：** 明明装了 Skill，但龙虾没有用
- **解法：** 检查触发词是否匹配你的用词习惯；在 AGENTS.md 里显式写「遇到 X 情况用 Y Skill」

##### 坑 3：定时任务重复执行
- **现象：** 同一个提醒发了好几次（比如早上心跳触发了三次）
- **解法：** 在 memory/heartbeat-state.json 里记录执行状态，任务执行前先检查是否已执行

##### 坑 4：在群聊里龙虾太话痨
- **现象：** 把龙虾加进群，它对每条消息都回复
- **解法：** 在 AGENTS.md 里明确「群聊场景只在被@或能贡献价值时回复，其他情况静默」

##### 坑 5：每次都要重新解释背景
- **现象：** 换一个 session 或者隔几天再聊，它完全不记得上次的内容
- **解法：** 这是记忆没配好。关键决策、项目背景要明确让它写入 MEMORY.md，不能指望它自动记

## 进阶玩法：多 Agent 协作

如果单只龙虾处理不过来，可以派多只虾分工合作。

适合场景：
- 一项任务需要「搜索资料 → 整理分析 → 生成文档 → 发送通知」多步骤
- 不同任务需要不同的专业背景或工具权限

基本模式： 主 Agent 负责决策和调度，子 Agent 负责具体执行。主 Agent 通过 sessions_spawn 派发任务，子 Agent 完成后汇报结果，这样session上下文不乱串，学会之后会自动起子agent干活。魔法提示：通过sessions_spawn起subAgent并行干xxx任务，干完回来汇报。

这是团队协作场景下的终极形态，但建议先把单 Agent 用熟了再上。还有通过多channel机器人对接多个Agent的玩法，给他们拉群，互相讨论派活，但实测会很乱、效率反而打折扣。


## 一句话总结
调教龙虾的本质，是建立一套可持续改进的工作关系：你越清楚地告诉它你是谁、你想要什么、什么是好答案，它就越能成为真正的工作伙伴，而不只是一个聪明的搜索框。

从 SOUL.md 开始，一步一步来。今天花 30 分钟配好，之后每次对话都省时间。
